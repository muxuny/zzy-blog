# 协作说明

这份文档用于后续新对话快速对齐项目习惯。

## 先看什么

1. `README.md`
2. 本文件
3. 当前任务对应的 `docs/superpowers/` 计划或设计文档

## 提交规范

- `git commit` 提交说明统一使用中文。
- 需要保留的改动要真正提交，不要只停留在聊天里。
- 提交前先确认 `git status`，不要把无关文件一起提交。

## 命名规范

- 文件名尽量使用中文，尤其是文档、说明和计划类文件。
- 代码文件如果已经有稳定引用，就不要为了中文名强行改动。

## 工作区规范

- 不要擅自回滚别人的未确认改动。
- 数据库改动优先写增量迁移脚本。
- 需要长期保留的约定，优先写进 `README.md` 或本文件。

## 启动约定

- 后端启动命令：`cd blog-backend && mvn.cmd spring-boot:run`
- 前端启动命令：`cd blog-frontend && npm.cmd run dev -- --host 127.0.0.1`
