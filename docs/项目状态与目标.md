# 项目状态与目标

这份文档记录博客项目的当前阶段、已完成功能、下一步目标和已排期功能。每次功能方向变动、功能完成或新增/删减时，需要同步更新这份文档。

## 项目定位

个人博客系统，包含公开阅读、创作中心和后台管理，目标是提供清晰、克制、内容优先的阅读和写作体验。

## 已完成功能

### 前端视觉系统

- 内容工作台视觉系统（主题变量、Element Plus 覆盖、全局样式）
- 亮色/暗色主题切换
- 响应式布局（桌面端、平板端、手机端）

### 公开阅读面

- 首页：搜索框（关键词按标题和摘要搜索）、话题导航（标签筛选按钮，点击后在首页内直接过滤）、最近更新（从当前文章列表取第一篇）、文章列表（同一位置根据筛选条件显示不同内容）、状态条、页脚
- 文章详情页：封面图、字数统计、预计阅读时间、目录高亮、悬浮回顶、上一篇/下一篇、相关文章推荐
- 文章收藏：文章详情页支持收藏和取消收藏；未登录收藏意图在登录后续接并返回原文章；“我的收藏”支持关键词搜索、标签筛选和分页，可公开文章支持整卡进入详情且卡内信息不单独高亮；不可用文章保留收藏时标题快照并显示统一不可用提示
- 标签文章列表页（/tag/:name）：独立页面，入口为文章卡片上的标签链接，点击后跳转到该标签下的所有文章；文章卡片标签已有 hover/focus 反馈
- 登录页、注册页

### 创作中心

- 我的文章列表：状态筛选（草稿/待审核/已发布/已驳回）、分组筛选、可见性筛选（公开/仅自己可见）
- 我的文章预览页：从“我的文章”列表标题进入，任何状态的文章都进入创作者侧预览；预览页支持目录导航，并提供直接进入编辑页的按钮；公开查看、分组、删除等动作保留在操作列
- 写文章/编辑文章：标题、摘要、封面图、标签选择、分组选择、可见性设置、Markdown 编辑器；编辑器标题下拉会随光标所在段落同步，代码块后会保留可继续输入的段落落点
- 文章状态流转：草稿或已驳回文章可提交为待审核，待审核文章可撤回为草稿，也可由管理员审核为已发布或已驳回；已发布文章编辑后会重新进入待审核
- 文章分组管理：创建、重命名、删除分组，调整文章分组

### 后台管理

- 仪表盘：文章状态分布（草稿/待审核/已发布/已驳回）、审核队列（待审核文章、待审核用户）、标签概览
- 文章管理：全站文章列表、按状态筛选、按可见性筛选、审核通过/驳回（需填写驳回原因）、编辑、删除
- 标签管理：创建、删除标签
- 图片管理：上传图片、删除图片
- 用户管理：用户列表、审核通过、禁用
- 个人资料：查看用户信息

## 当前阶段目标

继续做阅读连续性增强，让读者在博客内有更自然的阅读路径和回访能力。

### 优先级 1：我的阅读与阅读历史

- 已确认采用“双空间”结构：公开博客首页负责发现内容，“我的阅读”整合上次阅读、最近阅读和收藏，创作中心继续负责内容生产
- 第一阶段只记录登录用户成功打开的公开文章；同一用户与文章合并记录首次阅读时间、最近阅读时间和阅读次数
- 新增“我的阅读”概览和完整历史页；历史按时间分组，支持分页、删除单条和清空全部
- 文章后来变为不可公开时保留标题快照，记录置灰、禁止进入并提示文章暂未公开
- “我的阅读”只提供轻量的“发现更多文章”入口，不复制公开首页的文章流、搜索和标签筛选
- 第一阶段不在详情页展示“已读”标签，避免展示打开详情后必然成立的无效状态

## 已延期或暂不做

- 评论（需要评论表、审核机制，成本较高）
- 点赞（需要点赞关系表）
- 阅读趋势统计（需要后端统计接口）
- 热门排行（需要后端统计接口）
- 游客本地阅读历史
- 保存与恢复文章滚动位置、阅读进度
- 公开首页文章卡片的“已读”状态

## 当前约束

- 不引入新框架，继续使用 Vue 3 + Element Plus + Spring Boot + MyBatis-Plus
- 数据库变更优先使用增量脚本，避免直接重建数据库
- 保持现有视觉方向（内容工作台风格）
- 文件名尽量使用中文，尤其是文档和说明类文件
