# 我的阅读与阅读历史设计

## 背景

项目已经具备公开文章发现与阅读、文章详情连续阅读、登录注册、创作中心、后台管理和文章收藏能力，但当前只记录文章总浏览量，没有登录用户维度的阅读历史。

现有收藏实现已经提供可复用的基础模式：用户与文章关系表、标题快照、逻辑删除、不可公开文章保留记录、分页列表和整项点击交互。本功能在此基础上增强阅读连续性，同时避免把“我的文章”“我的收藏”“阅读历史”继续做成三个并列且形态相同的菜单列表页。

## 目标

- 为登录用户记录公开文章阅读历史。
- 新增“我的阅读”空间，聚合上次阅读、最近阅读和最近收藏。
- 提供按时间组织的完整阅读历史，以及单条删除和清空能力。
- 保留不可公开文章的标题快照，让用户知道历史记录为何不可访问。
- 保持公开博客首页负责内容发现，创作中心负责内容生产，不复制现有能力。

## 非目标

本期不实现以下能力：

- 游客阅读历史或浏览器本地历史。
- 保存滚动位置、阅读进度或恢复上次阅读位置。
- 公开首页文章卡片的“已读”标记。
- 文章详情页的“已读”标签；文章成功打开后必然成为已读，该标签在详情页没有足够信息价值。
- 阅读趋势统计、个性化推荐或热门排行。
- 在“我的阅读”中复制公开首页的文章流、搜索和标签筛选。

## 信息架构

采用“双空间”结构：

- 公开博客首页负责发现内容，游客和登录用户均可访问。
- “我的阅读”负责管理登录用户的阅读连续性，包括上次阅读、历史和收藏。
- 创作中心继续负责我的文章、文章分组、预览和编辑。
- 管理员后台继续保持独立入口。

登录用户菜单保留“写文章”快捷操作，并将原“我的文章”“我的收藏”核心入口整理为“我的阅读”和“创作中心”。管理员仍显示“后台管理”。游客导航不显示“我的阅读”和“创作中心”。

## 页面与交互

### 我的阅读概览

新增 `/reading` 鉴权路由，页面由三个不同形态的区域组成，不使用单一文章列表贯穿全页：

1. “上次阅读”突出最近一篇仍可公开访问的文章，展示封面、标题、摘要、作者和最近阅读时间。点击整项进入现有公开文章详情页。该入口表示重新打开最近文章，不承诺恢复滚动位置。
2. “最近阅读”以紧凑时间线展示最近 5 条记录，包括不可公开记录，并提供“查看全部历史”入口。
3. “最近收藏”展示最近 4 篇收藏文章，复用现有收藏数据与不可用规则，并提供“查看全部收藏”入口。

页面提供一个轻量的“发现更多文章”入口，跳转公开博客首页，不在概览中复制文章流、搜索或标签筛选。没有对应数据时，各区域独立显示空状态和可执行入口。

桌面端采用清晰的主次分区；移动端依次显示上次阅读、最近阅读和最近收藏。页面沿用当前内容工作台视觉变量，不嵌套卡片。可用文章整项可点击，悬停只强调外圈，不单独高亮标题、作者或标签。

### 完整阅读历史

新增 `/reading/history` 鉴权路由，默认每页 10 条，按 `last_read_at DESC, id DESC` 排序。当前页数据在前端按用户本地时间分为“今天”“昨天”“更早”。

每条记录展示封面缩略图、标题、作者、首次阅读时间、最近阅读时间和阅读次数。可用记录整项点击进入公开文章详情；删除图标按钮阻止卡片导航，并提供明确提示。

不可公开记录使用标题快照，整体置灰且不可点击，悬停或聚焦时提示“该文章暂未公开”。此时不返回正文、摘要、标签、封面或作者等当前文章信息。

页面支持删除单条和清空全部。清空操作必须二次确认。删除后重新加载当前页；若当前页已经为空且不是第一页，则回到上一页，保证分页不滞留在空页。

### 收藏兼容

现有 `/favorites` 路由继续有效，并作为“我的阅读”的收藏明细页使用，避免已有书签和登录回跳失效。本期不重写收藏筛选、分页和卡片交互。

## 数据模型

新增 `article_reading_history` 表。

业务字段：

- `user_id BIGINT NOT NULL`：阅读用户。
- `article_id BIGINT NOT NULL`：被阅读文章。
- `title_snapshot VARCHAR(200) NOT NULL`：用户最近一次成功阅读公开文章时的标题。
- `first_read_at DATETIME NOT NULL`：本次有效历史生命周期的首次阅读时间。
- `last_read_at DATETIME NOT NULL`：最近阅读时间。
- `read_count INT NOT NULL DEFAULT 1`：本次有效历史生命周期的阅读次数。

公共字段必须完整包含：

- `id BIGINT NOT NULL`
- `created_by VARCHAR(50)`
- `created_at DATETIME`
- `updated_by VARCHAR(50)`
- `updated_at DATETIME`
- `deleted TINYINT(1) NOT NULL DEFAULT 0`
- `version INT NOT NULL DEFAULT 0`

`created_at` 和 `updated_at` 只承担审计含义，不映射为首次或最近阅读时间。表建立 `(user_id, article_id)` 唯一索引、按用户和最近阅读时间查询的联合索引，以及按文章查询的索引。用户和文章外键沿用现有关系表约定。

同一用户再次读取同一篇有效历史中的文章时，原子更新 `title_snapshot`、`last_read_at`、`read_count`、更新审计字段和版本号，保留 `first_read_at`。用户删除历史后再次阅读时，重新激活原关系行，将 `first_read_at` 和 `last_read_at` 重置为当前时间，`read_count` 重置为 1；物理创建审计字段继续保持其审计含义。

数据库变更只生成并提交 UTF-8 编码的增量脚本，同时同步初始化脚本；协作过程中不执行数据库脚本，由用户手动执行。

## 后端边界

### 服务职责

- `ArticleService` 继续只负责公开文章读取、浏览量和文章本身的业务。
- `ReadingHistoryService` 负责历史写入、分页、单条删除、清空和可用状态转换。
- `ReadingSpaceService` 只聚合上次阅读、最近历史和最近收藏，不承载历史写入规则。
- `FavoriteService` 继续维护收藏关系，并向概览提供最近收藏数据。

历史查询沿用收藏的批量摘要模式：先分页取得关系记录和可用性，再批量读取公开文章摘要，避免逐条查询文章详情。

### 接口

新增以下登录用户接口：

- `GET /api/my/reading/overview`
- `GET /api/my/reading/history?page=1&size=10`
- `DELETE /api/my/reading/history/{articleId}`
- `DELETE /api/my/reading/history`

概览响应包含：

- `lastRead`：最近一条可用历史，没有时为 `null`。
- `recentHistory`：最近 5 条历史，包含不可公开记录。
- `historyTotal`：有效历史总数。
- `recentFavorites`：最近 4 条收藏。
- `favoriteTotal`：有效收藏总数。

历史条目包含 `articleId`、标题和可用状态；可用时包含摘要、封面、作者和文章浏览量；同时包含 `firstReadAt`、`lastReadAt`、`readCount`。不可用时只返回标题快照、时间、次数、`available=false` 和统一不可用提示。

## 记录流程

公开文章详情接口保持游客可访问，并接收可选登录身份：

1. 先按现有规则校验文章已经发布且公开可见。
2. 正常读取文章、更新总浏览量并组装详情。
3. 请求存在有效登录用户时，以已读取文章的 ID 和标题记录历史。
4. 游客、创作中心预览、后台预览、草稿预览、加载失败和无权访问请求均不记录。

阅读自己已经公开的文章同样记录。刷新或再次进入文章详情视为一次新的阅读，更新最近阅读时间并将阅读次数加一。

阅读历史属于附加能力。历史写入失败时回滚本次历史写入并记录后端错误日志，但不能让公开文章详情失败，用户仍可阅读正文。

## 文章不可用规则

查询时仅将“未逻辑删除、状态为已发布、可见性为公开或兼容旧数据的空值”判定为可用。

- 上次阅读只选择可用文章，自动跳过不可公开历史。
- 最近历史和完整历史保留不可公开记录。
- 收藏继续使用现有相同规则。
- 不可公开记录只展示用户成功阅读时保存的标题快照，避免泄露后来变更的私有内容。
- 用户可以删除不可公开历史，也可以通过清空操作删除全部有效历史。

## 异常与状态处理

- 概览通过单个聚合接口加载；请求失败时保留页面结构并展示统一重试入口。请求成功后，上次阅读、最近历史和最近收藏分别处理自己的有数据与空数据状态。
- 完整历史加载失败时保留页面结构，展示重试操作。
- 删除单条或清空失败时保留现有数据并提示失败，不做虚假成功更新。
- 删除操作由后端按当前登录用户限定条件，不能通过文章 ID 删除其他用户记录。
- 分页参数约束与现有收藏一致：页码不小于 1，每页数量为 1 到 100。
- 直接访问鉴权路由时，沿用现有登录页 `redirect` 参数，登录成功后返回原目标页。

## 测试与验收

后端测试覆盖：

- 游客读取文章不记录历史。
- 登录用户首次、重复读取和删除后重新读取。
- 文章详情加载失败时不新增历史。
- 历史写入失败时文章详情仍正常返回。
- 用户数据隔离、分页参数、单条删除和清空。
- 不可公开文章只返回标题快照和不可用状态。
- 上次阅读跳过不可公开文章，最近历史仍保留该记录。
- 概览正确聚合历史与收藏数量和条目上限。

前端测试覆盖：

- “我的阅读”和完整历史路由需要登录。
- 概览三个区域的数据、空状态和局部错误状态。
- 历史时间分组、分页、单条删除和清空确认。
- 可用历史整项导航，不可用历史禁止导航。
- 删除图标不会触发文章导航。
- 原 `/favorites` 路由和收藏交互保持正常。

实现完成后运行后端测试、前端测试和前端构建。协作过程不启动前后端服务，由用户自行启动并进行桌面端、移动端和真实数据库验证。

## 后续优化

- 游客端基于浏览器本地存储的阅读历史。
- 保存并恢复文章滚动位置。
- 更细的阅读进度和完成状态。
- 在公开首页文章卡片展示有实际价值的“已读”状态。
- 阅读趋势统计和个性化推荐。
