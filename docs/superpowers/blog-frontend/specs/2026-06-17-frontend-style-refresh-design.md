# 前端整体样式优化设计

日期：2026-06-17
项目：`blog-frontend`
状态：已确认，可进入实现计划

## 目标

将这个博客前端整体优化成一套统一的产品视觉系统。本次优化同时覆盖公开阅读页面和后台管理页面，让应用看起来像一个持续维护的完整产品，而不是默认组件样式与零散页面样式的拼接。

已确认的方向是 **内容工作台**：清晰、克制、内容优先，既适合阅读博客文章，也适合长期管理文章、标签、图片和用户。

## 当前情况

前端使用现有的单页应用框架、构建工具、路由、状态管理、组件库、正文渲染和代码高亮方案。现有样式比较薄：

- `src/styles/theme.css` 只定义了少量背景、文字、卡片、代码块和边框变量。
- `src/styles/global.css` 只有基础重置和简单文章正文样式。
- 大多数页面依赖局部样式和组件库默认样式。
- 公开页面与后台页面还没有统一的视觉系统。
- 多个可见中文文案已经乱码，需要在样式优化时一并恢复。

## 设计方向

应用整体应呈现“内容工作台”的气质：

- 阅读页面安静、舒展、便于浏览。
- 后台页面适合重复工作，信息密度更高，但不拥挤。
- 组件在间距、圆角、阴影、边框和色彩上保持一致。
- 现有组件库继续作为界面组件基础，但默认观感需要被统一和弱化。
- 动效保持克制，只服务于反馈和状态变化。

## 视觉系统

### 颜色变量

采用偏冷静的中性色，并用少量强调色建立操作感。

- 应用背景：`#F8FAFC`
- 面板背景：`#FFFFFF`
- 主要文字：`#172334`
- 次要文字：`#667085`
- 边框：`#D9E1EC`
- 主操作色：`#2F80ED`
- 辅助强调色：`#2FB3A3`
- 警告色：`#F59E0B`
- 危险色：`#E5484D`

暗色模式避免使用纯黑：

- 暗色应用背景：`#0F172A`
- 暗色面板背景：`#182235`
- 暗色浮层背景：`#1F2A3D`
- 暗色主要文字：`#E5EDF7`
- 暗色次要文字：`#A9B6C7`
- 暗色边框：`#2B3850`

### 字体与排版

不引入外部字体依赖，使用稳定的系统字体栈：

- 界面与文章正文：`-apple-system`、`BlinkMacSystemFont`、`Segoe UI`、`PingFang SC`、`Microsoft YaHei`、`Roboto`、`sans-serif`
- 代码：`SFMono-Regular`、`Consolas`、`Liberation Mono`、`Menlo`、`monospace`

排版处理：

- 页面标题使用更清晰的字重和更稳定的行高。
- 文章正文优化行高、段落间距和标题节奏。
- 后台说明文字、元信息和表格辅助文本使用更小字号，但对比度要高于当前默认灰色。

### 记忆点

使用克制的“工作台轨道线”作为系统记忆点：

- 内容分组使用细分隔线和左侧导向线。
- 文章卡片和文章详情中的元信息保持统一结构。
- 后台指标卡使用轻量状态条。
- 侧栏导航的激活态使用小色条，而不是厚重色块。

这个细节能让系统更有秩序感，同时避免不必要的装饰。

## 公开前台范围

### 顶部导航

`AppHeader.vue` 调整为更轻的吸顶导航：

- 左侧为品牌入口。
- 右侧为主题切换、登录/注册，或用户下拉菜单。
- 使用一致的间距，并采用轻微透明或轻微抬升的背景。
- 优化聚焦态和悬停态。
- 恢复用户菜单和登录注册操作中的乱码中文。

### 主题切换

`ThemeToggle.vue` 保持为紧凑的图标按钮：

- 使用稳定的方形点击区域。
- 在可行处补充清晰的无障碍标签。
- 亮色和暗色模式下都要与顶部导航按钮风格一致。

### 首页

`Home.vue` 保留双栏阅读布局：

- 主栏展示文章卡片。
- 侧栏保留标签筛选概念，具体左右位置由当前布局适配决定。
- 标签区从默认卡片改为更易扫读的筛选面板。
- 分页保持居中，但视觉上更安静。
- 空状态文案恢复为正常中文。

### 文章卡片

`ArticleCard.vue` 作为前台核心内容组件进行优化：

- 标题更醒目，层级更清楚。
- 摘要拥有更舒服的行高和截断效果。
- 日期、作者、阅读数使用统一的元信息行。
- 封面图使用稳定尺寸和比例。
- 标签使用紧凑标签片。
- 移动端下封面图与文字自动纵向排列，不产生溢出。
- 恢复“阅读数”等乱码文案。

### 文章详情

`ArticleDetail.vue` 调整为更完整的阅读页：

- 标题更大，但保持受控宽度。
- 元信息和标签集中放在标题下方。
- 正文统一处理标题、段落、引用、图片、表格、行内代码和代码块。
- 加载骨架屏匹配优化后的正文宽度。
- 恢复乱码分隔符和标签文案。

### 标签文章页

`TagArticles.vue` 与首页内容列表保持一致：

- 为当前标签提供清晰的页面标题。
- 复用文章卡片的视觉语言。
- 恢复空状态文案。

### 登录与注册

`Login.vue` 和 `Register.vue` 调整为更精致的工具型页面：

- 居中认证面板，优化间距、标签和输入框。
- 背景跟随主题，可使用轻微工作台网格或面板分区。
- 注册提示改成正式提示块。
- 链接与主操作按钮使用统一样式。
- 恢复表单标签、校验提示、成功提示和说明文案中的乱码中文。

## 后台范围

### 后台布局

`components/admin/AdminLayout.vue` 定义后台工作台基调：

- 深蓝灰侧栏。
- 浅色内容区。
- 菜单激活态使用清晰的轨道指示。
- 顶部栏展示欢迎语、返回博客和退出登录操作。
- 恢复后台菜单和顶部栏中的所有乱码中文。

### 仪表盘

`views/admin/Dashboard.vue` 作为后台样式基准：

- 使用页面标题和紧凑指标网格。
- 指标卡展示名称、数值和轻量状态条。
- 避免过大的装饰性卡片。
- 恢复统计项中的乱码中文。

### 其他后台页面

第一阶段不重写每个后台工作流，而是通过全局组件库样式覆盖和共享布局样式统一以下元素：

- 表格
- 表单
- 按钮
- 弹窗
- 卡片
- 分页
- 标签
- 上传控件
- 空状态

现有业务逻辑和接口行为保持不变。

## 实现边界

本次范围内：

- `src/styles/theme.css`
- `src/styles/global.css`
- `src/components/AppHeader.vue`
- `src/components/ThemeToggle.vue`
- `src/components/ArticleCard.vue`
- `src/views/Home.vue`
- `src/views/ArticleDetail.vue`
- `src/views/TagArticles.vue`
- `src/views/Login.vue`
- `src/views/Register.vue`
- `src/components/admin/AdminLayout.vue`
- `src/views/admin/Dashboard.vue`
- 如确实影响整体统一性，可对其他后台组件做少量定向样式修正。

本次不做：

- 修改接口。
- 修改登录、权限和路由行为。
- 替换现有组件库。
- 引入新的界面框架或样式框架。
- 重写编辑器业务逻辑。
- 添加大规模装饰动画。
- 制作营销落地页或宣传型首屏。

## 响应式表现

公开前台：

- 顶部导航在移动端不能文字溢出。
- 首页双栏在小屏下收为单栏。
- 文章卡片在小屏下封面图与文字纵向排列。
- 文章详情保持舒适的阅读宽度。

后台：

- 第一阶段保留固定侧栏模型。
- 常见笔记本宽度下内容不能明显破版。
- 表格横向溢出交给现有组件库能力处理。

## 可访问性与交互

- 链接、按钮、输入框和下拉触发器保持可见键盘聚焦态。
- 色彩对比度要强于当前偏淡的灰色文本。
- 只在图标足够熟悉且标签清楚时使用纯图标控件。
- 保留现有主题切换和暗色模式状态管理。
- 悬停态不能导致布局跳动。

## 风险

- 乱码中文可能不只存在于核心页面。实现时需要搜索明显可见的乱码文案并修复，同时避免改动无关逻辑。
- 组件库默认样式覆盖需要谨慎控制选择器范围，避免引入前台或后台回归。
- 暗色模式需要统一更新变量；零散的局部颜色可能在暗色下过亮或过暗。
- 本地数据未必包含真实文章封面图，因此文章卡片需要同时检查有图和无图状态。
