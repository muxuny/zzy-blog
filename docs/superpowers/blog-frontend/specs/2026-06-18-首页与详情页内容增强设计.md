# 首页与详情页内容增强设计

## 背景

项目的核心功能已经具备：公开文章列表、文章详情、标签筛选、登录注册、后台写作和审核流程。当前主要问题是公开侧页面信息层次不足：博客首页只有标签筛选和文章列表，文章详情页只有基础标题、元信息和正文，页面显得偏空，缺少个人博客应有的识别度、导览能力和阅读辅助。

本设计把优化拆成两个阶段，先做现有数据可以支撑的前端增强，再做需要后端补充数据或业务表的功能扩展。

## 设计目标

1. 让首页更像一个完整的个人博客入口，而不是单纯列表页。
2. 让详情页更适合阅读长文，提供目录、阅读信息和辅助导航。
3. 第一阶段不新增数据库表，不新增后端接口，优先使用已有文章、标签和用户相关数据。
4. 第二阶段再扩展需要后端支持的互动和推荐能力。
5. 继续保持当前项目的克制、清晰、偏内容阅读的视觉方向。

## 阶段一：公开侧内容增强

阶段一只使用现有接口和前端能力，目标是尽快让首页和详情页饱满起来。

### 首页增强

1. 顶部个人介绍区

首页顶部增加一个克制的个人介绍区，包含：

- 博客名称
- 一句简短说明
- 写作方向或关键词
- 文章总数和标签总数等轻量数据

它不做营销式大横幅，也不使用夸张背景。它的作用是给访问者建立“这是一个什么博客”的第一印象。

2. 搜索框

使用现有文章列表接口的 `keyword` 参数，增加文章标题搜索能力。搜索框放在内容区上方或介绍区下方，输入后可以按回车搜索，也可以点击按钮搜索。

3. 最新文章强调区

从当前文章列表中取最新一篇作为“最近更新”展示，视觉上比普通文章卡片更突出。其余文章仍以列表方式展示。

4. 标签区优化

当前标签区保留，但增加：

- “全部文章”按钮
- 当前筛选状态
- 标签数量说明

标签仍然是筛选入口，不展示标签下文章数量，因为当前公开接口没有直接提供这项统计。

5. 空状态优化

当没有文章或搜索无结果时，空状态需要说明原因：

- 没有公开文章：提示“还没有公开文章”
- 搜索无结果：提示“没有找到匹配文章”

6. 页脚

首页底部增加一个轻量页脚，包含项目名、版权年份和一句个人签名或说明，让页面有明确收尾。

### 详情页增强

1. 文章封面

如果文章存在封面图，在标题区下方或正文前展示。没有封面时不占位。

2. 文章信息侧栏

在桌面端展示文章信息侧栏，包含：

- 作者
- 发布日期
- 阅读数
- 字数
- 预计阅读时间
- 标签

移动端把侧栏信息折叠到正文前，避免挤压正文。

3. Markdown 目录

从文章 Markdown 内容中提取二级和三级标题，生成目录。桌面端目录放在右侧或文章旁边，移动端放在正文前。点击目录项滚动到对应标题。

4. 回到顶部

长文阅读时增加回到顶部按钮，出现在详情页右下角或正文末尾附近。它不需要后端支持。

5. 阅读辅助信息

通过前端计算文章字数和预计阅读时间。中文按字符数粗略计算，英文按单词数粗略计算，目标是提供参考，不追求精确统计。

## 阶段二：内容推荐和互动能力

阶段二需要后端配合，适合在第一阶段稳定后继续开发。

### 上一篇 / 下一篇

在文章详情页底部展示上一篇和下一篇公开文章。推荐由后端根据发布时间或文章 ID 返回，避免前端仅靠当前分页数据推算导致不准确。

### 相关文章

根据当前文章标签推荐其他公开文章。推荐规则可以先简单实现为“拥有相同标签的最新文章”，后续再考虑阅读数、发布时间等权重。

### 评论

恢复或重新设计评论功能。需要评论表、评论接口、审核或删除机制。未登录用户是否可评论需要单独确认。

### 点赞 / 收藏

如果要做用户互动，需要新增点赞或收藏关系表，并决定是否只允许登录用户操作。该能力不放入第一阶段。

### 阅读趋势和排行

如果后续需要热门文章、阅读排行、访问趋势，需要在后端提供更明确的统计接口。

## 页面结构建议

### 首页结构

从上到下：

1. 顶部介绍区
2. 搜索和筛选状态
3. 两栏布局：左侧标签筛选，右侧最新文章和文章列表
4. 分页
5. 页脚

移动端改为单列：

1. 顶部介绍区
2. 搜索框
3. 标签横向换行
4. 最新文章
5. 文章列表
6. 页脚

### 详情页结构

桌面端：

1. 返回按钮
2. 文章标题区
3. 封面图
4. 正文和侧栏双栏布局
5. 正文底部操作区

移动端：

1. 返回按钮
2. 标题区
3. 封面图
4. 文章信息
5. 目录
6. 正文
7. 回到顶部

## 视觉方向

整体继续使用当前已优化过的浅色、暗色主题变量，不引入新的大色系。视觉重点放在信息结构，而不是装饰。

可识别的设计元素是“阅读索引”：详情页用目录、字数、阅读时间、标签这些阅读辅助信息围绕正文展开，让页面看起来像一个真正服务阅读的个人知识库。

首页的视觉重点是“最近更新”：最新文章比普通列表更突出，但不做夸张大图首页，避免个人博客变成营销落地页。

## 数据和接口约束

第一阶段使用现有能力：

- `GET /api/articles`：公开文章列表，支持分页、标签和关键字。
- `GET /api/articles/{id}`：公开文章详情。
- `GET /api/tags`：标签列表。

第一阶段不新增后端字段、不新增表、不新增接口。

## 验证方式

第一阶段实现后需要验证：

1. 首页搜索、标签筛选、分页能组合使用。
2. 搜索无结果和无公开文章时空状态正确。
3. 详情页有标题时能生成目录，无标题时不显示空目录。
4. 详情页封面、标签、作者、阅读数缺失时页面不塌陷。
5. 明暗主题下首页和详情页文字、标签、边框可读。
6. 桌面端、平板端、手机端布局不重叠。
7. 前端生产构建通过。
