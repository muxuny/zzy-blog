# 文章收藏功能设计

## 背景

博客已经具备首页发现、文章详情、上一篇/下一篇和相关文章推荐，但登录读者还不能保存希望再次阅读的文章。当前阶段先实现边界清晰的文章收藏，不把阅读历史、已读状态或归档能力混入同一数据模型。

## 设计目标

1. 登录用户可以收藏和取消收藏公开文章。
2. 未登录读者发起收藏后，登录成功可以返回原文章并自动完成收藏。
3. 提供“我的收藏”页面，支持关键词搜索、标签筛选、分页和取消收藏。
4. 收藏后的文章如果暂时不再公开，读者仍能看到收藏时的标题和明确原因，不会误以为收藏无故丢失。
5. 收藏关系与阅读历史保持独立，避免不同生命周期和写入频率互相影响。

## 本次范围

### 包含

- 文章详情页收藏按钮和状态反馈。
- 未登录收藏意图的登录续接。
- 我的收藏页面。
- 按收藏时间倒序分页。
- 标题或摘要关键词搜索。
- 标签筛选。
- 不可公开文章的标题快照占位。
- 收藏和取消收藏的幂等处理。

### 不包含

- 已读/未读状态。
- 稍后读归档或自定义排序。
- 收藏分组、批量操作和导出。
- 收藏数、排行榜和统计分析。
- 点赞、评论和阅读历史。

## 数据模型

新增 `article_favorite` 表，实体遵循现有公共字段规范并继承 `BaseEntity`。

业务字段：

- `user_id`：收藏用户 ID，使用稳定用户主键隔离收藏关系。
- `article_id`：被收藏文章 ID。
- `title_snapshot`：收藏或恢复收藏时公开可见的文章标题。

公共字段：

- `id`
- `created_by`
- `created_at`
- `updated_by`
- `updated_at`
- `deleted`
- `version`

约束和索引：

- 唯一约束：`user_id + article_id`。
- 列表索引：`user_id + deleted + created_at`。
- 文章反查索引：`article_id + deleted`。
- `user_id` 和 `article_id` 分别关联用户表和文章表，物理删除用户或文章时级联清理收藏关系。

`deleted` 不进入唯一约束，避免同一用户反复收藏和取消后产生多条历史记录。取消收藏使用逻辑删除；重新收藏通过原子“新增或恢复”操作复用原记录。

状态规则：

1. 从未收藏：新增关系，保存当前标题快照和公共字段。
2. 已收藏：幂等成功，不改变收藏时间、标题快照和排序。
3. 已取消：恢复原记录，更新标题快照，把收藏时间重置为当前时间，并更新审计字段和版本。
4. 并发重复收藏：由唯一约束和原子新增/恢复语句保证只存在一条关系。

数据库变更同时写入：

- `blog-backend/src/main/resources/db/init.sql`
- `blog-backend/src/main/resources/db/migration/2026-07-16-新增文章收藏.sql`

增量脚本只新增表和索引，不删除或重建现有数据。

## 后端边界

### 收藏领域

新增独立的收藏实体、Mapper、Service 和 Controller。`FavoriteService` 负责：

- 将当前认证用户名解析为用户 ID。
- 收藏、恢复和取消收藏关系。
- 查询当前用户收藏状态。
- 按收藏时间、关键词和标签分页查询收藏关系。
- 生成可用和不可用两类收藏列表项。

### 文章领域

文章是否可收藏、是否能返回完整信息，继续遵循现有公开文章规则：

- 状态为 `published`。
- 可见性为 `public`；兼容历史空可见性数据。
- 未被逻辑删除。

`ArticleService` 提供批量获取公开文章摘要的能力，集中装配标签和作者，收藏服务不复制公开文章判断。

### 用户边界

所有收藏接口从认证上下文读取当前用户，不接受客户端传入用户 ID。普通用户和管理员都可以使用收藏能力，但只能读取和修改自己的关系。

## 接口设计

所有接口位于现有认证边界 `/api/my/**` 下。

### 收藏列表

```text
GET /api/my/favorites?page=1&size=10&keyword=&tagId=
```

规则：

- 按 `created_at` 倒序、关系 ID 倒序稳定分页。
- 公开文章的关键词匹配当前标题或摘要。
- 不可公开文章的关键词只匹配标题快照。
- 标签筛选只返回当前公开且匹配标签的文章。
- 默认列表和无标签关键词搜索可以包含不可公开文章。

返回条目：

```json
{
  "articleId": "758902345678901401",
  "title": "收藏时或当前文章标题",
  "summary": null,
  "coverImage": null,
  "authorName": null,
  "tags": [],
  "viewCount": null,
  "favoritedAt": "2026-07-16T12:00:00",
  "available": false,
  "unavailableMessage": "该文章暂未公开"
}
```

公开文章返回当前完整摘要信息；不可公开文章只返回关系中的文章 ID、标题快照、收藏时间、可用状态和统一提示。

### 收藏状态

```text
GET /api/my/favorites/{articleId}/status
```

返回当前用户是否存在未删除的收藏关系。

### 收藏文章

```text
PUT /api/my/favorites/{articleId}
```

只允许收藏当前可公开访问的文章。首次收藏、恢复收藏和重复收藏都返回成功；重复收藏不改变收藏时间。

### 取消收藏

```text
DELETE /api/my/favorites/{articleId}
```

只处理当前用户的关系。重复取消保持幂等并返回成功。

## 不可公开文章

文章收藏后可能被改为私密、撤回发布、逻辑删除或进入其他非公开状态。收藏列表保留该关系，但必须遵守以下信息边界：

1. 只展示收藏关系中的 `title_snapshot`，不读取或返回文章当前标题。
2. 不返回当前摘要、封面、标签、作者修改内容和具体状态。
3. 统一提示“该文章暂未公开”，不区分私密、撤回、审核或删除原因。
4. 列表项不可进入详情，但保留取消收藏操作。
5. 文章恢复公开后自动使用当前公开信息恢复正常列表项。

## 前端设计

### 文章详情页

标题区增加固定尺寸的收藏按钮，使用现有 Element Plus 图标和视觉变量。

按钮状态：

- 收藏
- 已收藏
- 处理中

登录用户在文章加载成功后查询收藏状态。收藏或取消请求期间禁用按钮，成功后更新状态并显示轻量反馈；收藏状态请求失败不影响正文阅读。

### 登录续接

未登录读者点击收藏时，跳转到登录页，并把当前文章地址和待收藏标记放入现有安全重定向参数。

登录成功返回详情页后：

1. 等待公开文章和认证状态加载成功。
2. 自动调用幂等收藏接口。
3. 无论成功或失败，都使用 `router.replace` 清除待收藏标记，避免刷新重复触发。
4. 失败时保留正常收藏按钮，用户可以手动重试。

### 我的收藏页面

新增需要登录的 `/favorites` 路由，并在用户下拉菜单增加“我的收藏”。

页面结构：

1. 页面标题和收藏数量。
2. 关键词、标签、搜索和重置组成的紧凑工具栏。
3. 收藏列表。
4. 分页。

收藏列表使用独立组件，不直接复用整张 `ArticleCard`：

- 可用文章展示封面、标题、摘要、作者、阅读数、标签和收藏时间。
- 不可用文章整体置灰，禁止详情跳转。
- 不可用标题悬停或键盘聚焦时提示“该文章暂未公开”。
- 所有条目保留固定尺寸的取消收藏图标按钮。
- 取消收藏不弹确认框，成功后从当前结果移除；如果当前页因此为空且不是第一页，回到上一页重新加载。

空状态区分：

- 尚未收藏文章。
- 当前关键词或标签筛选无结果。

移动端工具栏和列表项改为纵向排列，标题、提示和取消按钮不能重叠。

## 异常与并发

- 收藏不存在或当前不可公开的文章，提示“文章不存在或暂不可收藏”。
- 收藏和取消均为幂等操作，允许登录续接、重复点击和网络重试。
- 唯一约束防止并发请求创建重复关系。
- 列表加载失败时显示可重试错误，不用空状态掩盖异常。
- 不可用文章不暴露具体状态和当前私密内容。
- 后端分页参数继续使用现有默认值和分页能力。

## 测试策略

### 后端

- 数据库初始化和增量迁移包含公共字段、唯一键、索引和外键。
- 首次收藏保存标题快照。
- 重复收藏不改变收藏时间。
- 取消后重新收藏会恢复关系并更新时间和快照。
- 不可公开文章不能新收藏。
- 收藏状态和取消操作按当前用户隔离。
- 列表正确返回公开文章和不可用占位项。
- 不可用项不泄露当前私密字段。
- 关键词、标签、分页和稳定排序正确。
- Controller 路径和权限规则正确。

### 前端

- 未登录收藏重定向和登录后自动续接。
- 待收藏标记只消费一次并被清理。
- 收藏按钮状态和重复点击保护。
- 搜索、标签、重置和分页参数。
- 不可用项禁止详情跳转但可以取消收藏。
- 正常空状态、筛选空状态、错误和重试状态。
- 前端测试、生产构建和现有静态校验通过。
- 桌面端、移动端、亮色和暗色主题下没有遮挡或文本溢出。

## 完成标准

1. 登录用户能在文章详情页稳定收藏和取消收藏。
2. 未登录收藏意图能在登录后自动完成。
3. 我的收藏支持关键词、标签、分页和取消收藏。
4. 不可公开文章保留标题快照提示且不泄露当前私密内容。
5. 重复和并发请求不产生重复关系。
6. 数据库变更使用增量迁移并保留公共字段规范。
7. `docs/项目状态与目标.md` 在实现完成后同步收藏完成状态和下一阶段目标。
8. 前后端全量测试和前端生产构建通过。
