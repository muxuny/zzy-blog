# 后台仪表盘轻量可视化设计

## 背景

当前后台仪表盘只展示几个基础数字，页面过于空旷，无法帮助管理员快速判断内容、审核和标签的整体状态。现阶段项目已有数据以数量统计和状态分布为主，不需要引入专业图表库，优先使用现有接口和样式能力完成轻量可视化。

## 目标

1. 让管理员进入后台后能立刻看到内容运营状态。
2. 增强页面的信息密度，避免只有数字卡片。
3. 不新增后端接口，不引入图表库，降低实现和维护成本。
4. 保持后台工具风格：清晰、克制、便于扫描。

## 范围

本次只优化后台仪表盘页面：

- 文章总览指标
- 文章状态分布
- 待处理审核队列
- 标签概览
- 空数据和加载状态

不包含：

- 访问量统计
- 阅读排行
- 评论趋势
- 用户增长趋势
- 新增数据库字段
- 新增后端统计接口

这些能力需要后续补充真实统计数据后再设计。

## 数据来源

仪表盘继续复用现有前端接口：

- 文章管理列表接口：获取全部文章总数、不同状态文章数量、待审核文章列表。
- 标签列表接口：获取标签总数和标签名称。
- 用户管理列表接口：获取待审核用户数量和待审核用户列表。

如果某个接口请求失败，对应区域显示空状态或短提示，不阻塞整个仪表盘渲染。

## 页面结构

页面从上到下分为四个区域：

1. 顶部指标区

展示四个核心指标：

- 文章总数
- 已发布文章
- 待审核文章
- 待审核用户

每个指标卡片保留醒目的数字，同时增加小型说明和状态色，帮助区分不同类型的信息。

2. 内容状态区

展示文章状态分布：

- 草稿
- 待审核
- 已发布
- 已驳回

使用横向比例条和列表数字组合，不使用复杂图表。比例条表达总体分布，列表数字保证精确可读。

3. 审核队列区

展示需要管理员处理的内容：

- 最近待审核文章
- 待审核用户

每条记录展示名称、作者或账号信息、当前状态，并提供跳转入口。没有待处理事项时显示明确的空状态。

4. 标签概览区

展示标签总数和部分标签名称，使用轻量排行条或标签组表达内容组织情况。当前没有文章和标签关联统计接口，因此不展示“标签下文章数量排行”，避免假数据。

## 视觉方向

整体延续后台管理的沉稳风格，但增加可视化层次：

- 背景保持干净，重点区域使用浅色分隔和细边框。
- 指标数字更清晰，但不做夸张大标题。
- 状态颜色保持语义一致：已发布偏成功色，待审核偏警示色，驳回偏危险色，草稿偏中性色。
- 可视化条使用纯样式实现，支持明暗主题。
- 卡片圆角保持克制，避免营销页式大面积装饰。

一个小的视觉记忆点是“状态轨道”：文章状态分布用同一条轨道拆分成不同状态段，管理员可以一眼看到内容从草稿、审核到发布的比例。

## 交互设计

- 点击待审核文章，进入文章管理或编辑页面查看详情。
- 点击待审核用户，进入用户管理页面。
- 加载时每个区域独立显示骨架或轻量加载态。
- 数据为空时显示短文案，例如“暂无待审核文章”。
- 请求失败时不打断整个页面，只在对应区域显示“加载失败”。

## 响应式设计

- 桌面端：顶部四列指标，中间左右两列布局。
- 平板端：指标改为两列，内容状态和审核队列上下排列。
- 手机端：所有区域单列展示，比例条和队列项保持可读。

## 实现边界

前端只做数据聚合和展示，不改变业务规则：

- 不新增后端接口。
- 不新增第三方图表依赖。
- 不改变文章审核、用户审核、标签管理的原有流程。
- 不使用模拟统计数据填充真实业务含义。

## 验证方式

实现完成后需要验证：

1. 前端生产构建通过。
2. 仪表盘在有数据、无数据、接口失败时都能正常显示。
3. 明暗主题下状态标签和可视化条可读。
4. 桌面端和平板端布局不拥挤，手机端不出现文字重叠。
5. 已有后台路由和用户管理、文章管理页面不受影响。
