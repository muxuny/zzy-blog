# 首页与详情页视觉返工 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把首页和详情页从“文字堆叠”返工为有首屏、有视觉主体、有阅读层次的公开博客页面。

**Architecture:** 只调整前端 Vue 单文件组件的模板和样式，不新增后端接口、不改数据库。保留已经实现的搜索、标签筛选、分页、目录、字数和阅读时间功能。

**Tech Stack:** Vue 3、Element Plus、CSS Grid、CSS 变量、现有文章和标签接口。

---

### Task 1: 首页杂志式首屏

**Files:**
- Modify: `blog-frontend/src/views/Home.vue`

- [ ] 把顶部介绍区、搜索、最近更新合并为一个左右分栏的首屏模块。
- [ ] 左侧展示博客标题、说明、搜索框和话题按钮。
- [ ] 右侧展示最近更新主推卡片，包含封面图或设计化占位图、日期、作者、阅读数。
- [ ] 首屏底部展示公开文章数、标签数、当前筛选状态，形成更清晰的数据节奏。

### Task 2: 首页文章流

**Files:**
- Modify: `blog-frontend/src/views/Home.vue`

- [ ] 把标签筛选改为“话题导航”，避免像后台表单。
- [ ] 普通文章列表改成更有节奏的内容流：左侧标题摘要，右侧元信息或小封面。
- [ ] 保留分页和空状态。
- [ ] 移动端改为单列，搜索和话题按钮不溢出。

### Task 3: 详情页阅读封面

**Files:**
- Modify: `blog-frontend/src/views/ArticleDetail.vue`

- [ ] 标题区改为更像文章封面页：标题、摘要式元信息、标签、阅读时间集中呈现。
- [ ] 有封面图时做大图；无封面时使用结构化标题区，不出现空白占位。
- [ ] 正文和阅读侧栏保持双栏，但减少卡片堆叠感。

### Task 4: 详情页阅读辅助

**Files:**
- Modify: `blog-frontend/src/views/ArticleDetail.vue`

- [ ] 目录面板更像阅读索引，弱化边框和卡片感。
- [ ] 阅读信息变成紧凑指标条。
- [ ] 回到顶部按钮改成轻量悬浮/末尾操作，不显得像后台按钮。
- [ ] 移动端目录、阅读信息和正文之间保持清晰间距。
