# 文章搜索与状态文档基线对齐 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将文章关键词搜索统一为标题或摘要匹配，并让项目状态文档准确描述现有文章状态流转。

**Architecture:** 在现有 `ArticleServiceImpl` 中集中封装关键词查询条件，公开阅读、创作中心和后台管理三类列表复用同一规则。状态流转只更新当前状态文档，不修改已经完成的历史设计文档。

**Tech Stack:** Java 8、Spring Boot 2.7、MyBatis-Plus、JUnit 5、Mockito、AssertJ、Markdown

- [ ] **Step 3: 写入最小实现**

在 `ArticleServiceImpl` 增加并复用以下私有方法：

```java
private void applyKeywordFilter(LambdaQueryWrapper<Article> wrapper, String keyword) {
    if (keyword == null || keyword.isEmpty()) {
        return;
    }
    wrapper.and(keywordWrapper -> keywordWrapper
            .like(Article::getTitle, keyword)
            .or()
            .like(Article::getSummary, keyword));
}
```

公开列表、后台列表和 `selectMyArticlePage` 都调用该方法，删除三处仅匹配标题的重复代码。
