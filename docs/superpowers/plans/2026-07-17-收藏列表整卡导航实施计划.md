# 收藏列表整卡导航 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让“我的收藏”中的可公开文章支持整卡进入详情，同时保留独立的取消收藏操作和不可公开文章限制。

**Architecture:** 在 `FavoriteArticleItem.vue` 内为可公开卡片渲染覆盖整卡的 `RouterLink`，链接与取消收藏按钮保持同级，依靠层级隔离两个交互区域。收藏页面不再接收 `open` 事件或执行路由跳转；现有静态 UI 校验负责锁定链接语义、不可用分支、样式反馈和雪花 ID 路由规则。

**Tech Stack:** Vue 3、Vue Router 4、Element Plus、Node.js 静态契约校验、Vite

---

## 文件结构

- Modify: `blog-frontend/scripts/verify-favorite-ui.mjs`：定义收藏卡片整卡导航的自动校验契约。
- Modify: `blog-frontend/src/components/FavoriteArticleItem.vue`：渲染整卡链接、隔离取消收藏按钮并保持统一反馈。
- Modify: `blog-frontend/src/views/Favorites.vue`：移除已下沉到卡片组件的导航转发逻辑。
- Modify: `docs/项目状态与目标.md`：把收藏列表交互优化从进行中转为已完成。
