# 收藏列表整卡导航设计

## 背景

“我的收藏”当前只有可公开文章的标题可以进入详情。卡片已经在鼠标悬停时显示边框和阴影反馈，但正文摘要、封面、标签、元信息和留白区域不能点击，交互反馈与实际可点击范围不一致。

## 目标

- 可公开文章支持点击整张收藏卡片进入文章详情。
- 卡片内除“取消收藏”外的区域采用统一导航行为。
- 标题、摘要、封面、标签和元信息不增加单独的 hover 高亮，只保留整卡外圈反馈。
- 鼠标和键盘用户都能清楚、安全地完成导航。
- 不可公开文章继续置灰、不可跳转，并保留“该文章暂未公开”提示。

## 非目标

- 不把收藏列表中的标签改为独立筛选或跳转入口。
- 不调整卡片内容、排版、颜色体系和取消收藏流程。
- 不改变文章详情路由和后端接口。

## 交互设计

可公开卡片内部增加覆盖整卡的 `RouterLink`，目标地址为 `/article/{articleId}`。链接只承担导航和整卡焦点，不对卡片中的任何单项信息增加颜色、下划线或背景反馈。

“取消收藏”按钮与整卡链接保持同级，通过层级置于链接上方，点击时仍只执行取消收藏。键盘聚焦整卡链接时显示整卡轮廓；聚焦取消收藏按钮时沿用现有按钮反馈。

不可公开卡片不渲染整卡链接，因此鼠标和键盘都不能进入详情。标题快照继续支持聚焦并展示不可公开提示。

## 组件边界与数据流

- `FavoriteArticleItem.vue` 根据 `item.available` 决定是否渲染整卡路由链接。
- 文章 ID 按原值拼接路由，不进行数值转换，避免雪花 ID 精度丢失。
- 整卡链接直接完成前端路由导航，组件不再通过 `open` 事件把导航回传给收藏页面。
- `remove` 事件和收藏页面的取消收藏逻辑保持不变。

## 可访问性

- 整卡导航使用真正的链接语义，并提供包含文章标题的可访问名称。
- 链接支持 Tab 聚焦和 Enter 激活。
- 焦点轮廓覆盖整张卡片，避免只突出某段内容。
- 删除按钮不嵌套在链接内，避免交互控件嵌套。

## 验证

- 自动校验确认可公开卡片存在整卡路由链接，且路由保留原始文章 ID。
- 自动校验确认取消收藏按钮位于链接之外，并继续隔离点击行为。
- 自动校验确认不可公开卡片不生成导航链接，提示行为保持不变。
- 自动校验确认标题等卡内信息没有独立 hover 高亮。
- 运行前端测试和生产构建，检查现有收藏行为没有回归。
