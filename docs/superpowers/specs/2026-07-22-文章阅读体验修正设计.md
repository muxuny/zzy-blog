# 文章阅读体验修正设计

## 背景

登录用户阅读位置与进度已完成代码实现并进入待验收阶段。验收中发现两个体验问题：

- 文章详情页的“返回”按钮只在页面顶部。用户读到中段想返回时，需要先滚动回顶部，滚动动作会把阅读位置覆盖到顶部附近。
- “继续阅读”提示嵌在标题下方，视觉优先级不足，用户不仔细看容易错过。

## 目标

- 用户阅读到文章中段时，可以直接返回上一页或首页，不需要先回顶部。
- 有可恢复位置时，用更明显但可关闭的方式提示继续阅读。
- 不改变阅读位置后端保存规则，不新增数据库字段，不影响游客边界。

## 方案

文章详情页保留顶部返回按钮，同时在滚动超过现有阈值后显示清晰图标化阅读工具条。工具条固定在视口右下角，脱离右侧目录和阅读提示卡片，避免看起来像侧栏卡片的外挂操作。工具条使用接近原返回顶部按钮的白底浮层、浅描边和更明确的阴影，包含“返回”和“返回顶部”两个同尺寸图标按钮，依靠 tooltip 表达含义，避免文字胶囊和强渐变抢占正文注意力。点击“返回”前先触发一次强制保存，再执行原有 `goBack` 逻辑。

继续阅读提示从正文前的内联提示条调整为 `el-dialog` 弹窗。弹窗仅在 `showResumePrompt` 为真时出现，展示“上次读到 xx%”，提供“继续阅读”和“稍后再说”两个操作。点击继续阅读后关闭弹窗并跳转到保存位置；点击稍后再说后本次页面不再提示。

## 交互边界

- 低于 5% 的阅读位置不弹窗，沿用现有 `shouldShowResumePrompt` 判断。
- 文章内容更新导致 `canResume=false` 时不弹窗。
- 游客不显示弹窗，不保存阅读位置。
- 滚动中自动保存节流逻辑保持不变，离开页面前补保存保持不变。
- 悬浮工具条只解决中段离开和回顶入口，不新增确认框，避免打断阅读，也不跟随侧栏内容定位；常态保持白底清晰可见，hover 时提供轻量主题色反馈。

## 验证

- 前端源契约测试覆盖弹窗、继续阅读动作、关闭动作、清晰白底图标工具条、视口右下角定位、点击返回前补保存。
- 运行 `node --test src/utils/readingPosition.test.mjs`。
- 运行 `npm.cmd run build`。
